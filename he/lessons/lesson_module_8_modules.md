## דרישות קדם

מה התלמיד צריך לדעת לפני השיעור:

- [ ] פונקציות - הגדרה וקריאה
- [ ] טיפוסי נתונים בסיסיים: מחרוזות, מספרים, רשימות, מילונים
- [ ] זרימת בקרה: לולאות ותנאים

---

## מטרות למידה

בסוף השיעור, התלמיד יוכל:

1. לייבא מודולים מובנים באמצעות `import` ו-`from ... import`
2. להשתמש במודול `datetime` לעבודה עם תאריכים וזמנים
3. להשתמש במודול `math` לפעולות מתמטיות מתקדמות
4. להשתמש במודול `json` לשמירה וטעינת נתונים
5. לחקור מודולים חדשים באמצעות `dir()` ותיעוד

---

## מושגי מפתח

| מושג | הסבר במשפט אחד |
|------|----------------|
| מודול (Module) | קובץ שמכיל קוד פייתון (פונקציות, משתנים) שאפשר לייבא |
| `import` | מביא מודול לתוכנית שלך כדי שתוכל להשתמש בתוכן שלו |
| `from X import Y` | מייבא רק פריטים ספציפיים ממודול |
| ספרייה סטנדרטית | מודולים שמגיעים מובנים עם פייתון (לא צריך התקנה) |
| מרחב שמות (Namespace) | ה"מיכל" לפונקציות של מודול, ניגשים עם `module.function()` |

---

## תפיסות שגויות נפוצות

1. **שכחה לייבא**: ניסיון להשתמש בפונקציית מודול בלי לייבא קודם
   **המציאות**: חייבים לייבא בראש הקובץ לפני השימוש

2. **בלבול תחביר ייבוא**: ערבוב `import X` לעומת `from X import Y`
   **המציאות**: `import X` → משתמשים ב-`X.function()`, `from X import Y` → משתמשים ב-`Y()` ישירות

3. **חשיבה שכל מודול צריך התקנה**: הנחה שכל המודולים צריכים pip install
   **המציאות**: מודולי הספרייה הסטנדרטית כבר זמינים

4. **שימוש חוזר בשמות מודולים**: יצירת קובץ בשם `random.py` ושבירת ייבואים
   **המציאות**: לא לקרוא לקבצים שלכם בשמות של מודולים מובנים

## רצף ההוראה

### שלב 1: מה הם מודולים?
- להסביר: קוד שנכתב מראש שאפשר להשתמש בו בתוכניות שלכם
- להשוות לייבוא פונקציות כמו השאלת כלים
- להראות את שני סגנונות הייבוא ומתי להשתמש בכל אחד
- לשים לב ל: שמים משפטי import באמצע הקוד

### שלב 2: מודול random (חזרה)
- הם ראו את זה קודם - חזרה מהירה
- `random.randint()`, `random.choice()`, `random.shuffle()`
- חימום טוב לפני מודולים חדשים
- לשים לב ל: בדיקת ביטחון לפני שממשיכים

### שלב 3: מודול datetime
- `datetime.datetime.now()` - תאריך וזמן נוכחיים
- פורמט תאריכים עם `.strftime()`
- חישוב הפרשי זמן
- לשים לב ל: ה-datetime הכפול (datetime.datetime)

### שלב 4: מודול math
- `math.sqrt()`, `math.pow()`, `math.floor()`, `math.ceil()`
- קבועים: `math.pi`, `math.e`
- מתי להשתמש ב-math לעומת אופרטורים מובנים
- לשים לב ל: בלבול עם אופרטורים בסיסיים

### שלב 5: מודול json
- שמירת נתונים: `json.dump()` ו-`json.dumps()`
- טעינת נתונים: `json.load()` ו-`json.loads()`
- שימוש מעשי: שמירת מצב משחק, תצורה
- לשים לב ל: יסודות טיפול בקבצים (open, close)

### שלב 6: חקירת מודולים
- `dir(module)` - לראות מה זמין
- לקבל עזרה: `help(module.function)`
- קריאת תיעוד
- לשים לב ל: עומס מידע

## נקודות בדיקה

### אחרי שלב 1 (ייבואים)
לשאול: "מה ההבדל בין `import math` ו-`from math import sqrt`?"
צפוי: עם `import math` משתמשים ב-`math.sqrt()`, עם `from` משתמשים ב-`sqrt()` ישירות

### אחרי שלב 3 (datetime)
לשאול: "איך מקבלים את התאריך והזמן הנוכחיים?"
צפוי: `datetime.datetime.now()` או `from datetime import datetime` ואז `datetime.now()`

### אחרי שלב 5 (json)
לשאול: "באיזה מודול היית משתמש כדי לשמור מילון לקובץ?"
צפוי: json

### סוף השיעור
לשאול: "איך היית מגלה אילו פונקציות יש במודול?"
צפוי: `dir(module)`, תיעוד, או `help(module)`

## אם התלמיד מתקשה

- **אם מבולבל לגבי סגנונות ייבוא**: להישאר עם `import X` ותמיד להשתמש ב-`X.function()` לעקביות
- **אם מוצף מאפשרויות**: להתמקד במודול אחד בכל פעם, לשלוט בו, ואז להמשיך
- **אם JSON מבלבל**: להתחיל עם `json.dumps()` (למחרוזת) לפני פעולות קובץ
- **אם datetime מורכב**: להשתמש ב-`datetime.date.today()` לפעולות פשוטות יותר עם תאריך בלבד קודם
