## תוכן השיעור

### בנייה על חלק 1

בחלק 1 יצרנו פונקציות פשוטות שתמיד עושות בדיוק אותו דבר. אבל מה אם נרצה פונקציה שיכולה לעשות דברים קצת שונים בהתאם למה שניתן לה?

### להפוך פונקציות לגמישות

```python
# הפונקציה הזו תמיד אומרת "Hello!" - לא מאוד גמיש
def say_hello():
    print("Hello!")

# מה אם נרצה לברך אנשים שונים?
```

### הצגת פרמטרים

פרמטרים מאפשרים לפונקציה לקבל קלט:

```python
def greet(name):
    print(f"Hello, {name}!")

# עכשיו אפשר לברך כל אחד!
greet("{{hero}}")      # Hello, {{hero}}!
greet("{{heroine}}")   # Hello, {{heroine}}!
greet("{{friend}}")    # Hello, {{friend}}!
```

להסביר את המינוח:
- `name` ב-`def greet(name):` הוא **פרמטר** (מציין המקום)
- `"{{hero}}"` ב-`greet("{{hero}}")` הוא **ארגומנט** (הערך הממשי)

### פרמטרים מרובים

פונקציות יכולות לקבל מספר פיסות מידע:

```python
def greet_with_title(title, name):
    print(f"Hello, {title} {name}!")

greet_with_title("Professor", "{{mentor}}")  # Hello, Professor {{mentor}}!
greet_with_title("Dr.", "{{villain}}")       # Hello, Dr. {{villain}}!
```

הסדר חשוב! ארגומנטים מותאמים לפרמטרים לפי מיקום.

### דוגמה מעשית: ציור גמיש

```python
def draw_square(size):
    for i in range(4):
        t.forward(size)  # להשתמש בפרמטר!
        t.right(90)

# עכשיו אפשר לצייר ריבועים בגדלים שונים
draw_square(50)   # ריבוע קטן
draw_square(100)  # ריבוע בינוני
draw_square(200)  # ריבוע גדול
```

### לשים לב ל

- **בלבול מיקומי**: עם פרמטרים מרובים, לטעות בסדר
- **ארגומנטים חסרים**: לקרוא ל-`greet()` בלי לספק שם
- **בלבול בין פרמטר לארגומנט**: לזכור - פרמטר בהגדרה, ארגומנט בקריאה
