- לייבא מודולים מובנים באמצעות תחביר `import` ו-`from ... import`

---

## מושגי מפתח

| מושג | הסבר במשפט אחד |
|------|----------------|
| מודול (Module) | קובץ שמכיל קוד פייתון (פונקציות, משתנים) שאפשר לייבא |
| `import` | מביא מודול לתוכנית שלך כדי שתוכל להשתמש בתוכן שלו |
| `from X import Y` | מייבא רק פריטים ספציפיים ממודול |
| ספרייה סטנדרטית | מודולים שמגיעים מובנים עם פייתון (לא צריך התקנה) |
| מרחב שמות (Namespace) | ה"מיכל" לפונקציות של מודול, ניגשים עם `module.function()` |

## תוכן השיעור

### התחלה

להסביר שמודולים הם קוד שנכתב מראש שאפשר להשתמש בו בתוכניות שלכם - כמו להשאיל כלים מארגז כלים במקום לבנות אותם בעצמכם.

פייתון מגיע עם הרבה מודולים מובנים ב"ספרייה הסטנדרטית" שלו - לא צריך התקנה!

### שתי דרכים לייבא

**שיטה 1: לייבא את כל המודול**
```python
import math
result = math.sqrt(16)  # משתמשים ב-module_name.function()
```

**שיטה 2: לייבא פריטים ספציפיים**
```python
from math import sqrt
result = sqrt(16)  # משתמשים בפונקציה ישירות
```

### מתי להשתמש בכל סגנון

- להשתמש ב-`import module` כשצריכים הרבה דברים מהמודול, או רוצים בהירות לגבי מאיפה הפונקציות מגיעות
- להשתמש ב-`from module import item` כשצריכים רק דבר אחד או שניים ספציפיים

### לשים לב ל

- תלמידים ששמים משפטי import באמצע הקוד (ייבואים צריכים להיות בראש)
- יצירת קבצים עם אותו שם כמו מודולים מובנים (למשל, קריאה לקובץ `random.py`)
- שכחה לייבא לפני שימוש בפונקציית מודול

## נקודת בדיקה

לשאול: "מה ההבדל בין `import math` ו-`from math import sqrt`?"
צפוי: עם `import math` משתמשים ב-`math.sqrt()`, עם `from` משתמשים ב-`sqrt()` ישירות
