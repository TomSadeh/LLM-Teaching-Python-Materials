## תוכן השיעור

### הבעיה עם מילונים

להתחיל עם מילון שמייצג דמות:

```python
hero = {
    "name": "{{hero}}",
    "health": 100,
    "strength": 15
}
```

עכשיו להראות פונקציות שעובדות עם המילון:

```python
def attack(character, target):
    damage = character["strength"]
    target["health"] -= damage
    print(f"{character['name']} attacks for {damage} damage!")

def heal(character, amount):
    character["health"] += amount
    print(f"{character['name']} heals for {amount} health!")
```

### הבעיה

- הנתונים (מילון) והפונקציות נפרדים
- כל פונקציה צריכה לקבל את המילון כפרמטר
- שום דבר לא מונע להעביר את המילון הלא נכון
- ככל שתוכניות גדלות, זה הופך מבולגן ורגיש לשגיאות

### הפתרון: מחלקות

מחלקות מאפשרות לנו לאגד נתונים ופונקציות ביחד:

```python
class Character:
    # הנתונים (תכונות) והפונקציות (שיטות)
    # גרים ביחד במקום אחד
    pass
```

חשבו על זה כך:
- **מילון**: תיקייה עם דפים (נתונים) בתוכה
- **מחלקה**: תיקייה עם דפים והוראות מה לעשות איתם
