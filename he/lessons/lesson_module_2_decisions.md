- [ ] שימוש ב-`input()` לקבלת נתונים מהמשתמש
- [ ] מושגי השוואה בסיסיים (גדול, קטן, שווה)

## מטרות למידה

בסוף השיעור, התלמיד יוכל:

1. לכתוב משפטי `if` להרצת קוד מותנה
2. להשתמש ב-`else` לספק חלופה כשהתנאי שקרי
3. להשתמש ב-`elif` לבדיקת תנאים מרובים ברצף
4. לשלב תנאים עם `and`, `or`, ו-`not`
5. לקנן משפטי תנאי בעת הצורך

---

## מושגי מפתח

| מושג | הסבר במשפט אחד |
|------|----------------|
| `if` | מריץ את הקוד המוזח רק כשהתנאי הוא True |
| `else` | רץ כשתנאי ה-`if` הוא False - ה"אחרת" |
| `elif` | קיצור של "else if" - בודק תנאי נוסף אם הקודמים היו False |
| אופרטורי השוואה | `==` (שווה), `!=` (לא שווה), `<`, `>`, `<=`, `>=` |
| בוליאני (Boolean) | ערך שהוא או `True` או `False` |

---

## תפיסות שגויות נפוצות

1. **שימוש ב-`=` במקום `==`**: כתיבת `if x = 5:` במקום `if x == 5:`
   **המציאות**: `=` מקצה ערכים, `==` משווה אותם

2. **מספר `if` במקום `elif`**: שימוש במספר משפטי `if` במקום `elif`
   **המציאות**: מספר `if` כולם נבדקים; `elif` עוצר אחרי ההתאמה הראשונה

3. **שכחת הנקודתיים**: כתיבת `if x == 5` בלי ה-`:`
   **המציאות**: פייתון דורש נקודתיים אחרי התנאי

4. **הזחה אחרי if**: אי הזחת הקוד שצריך לרוץ מותנה
   **המציאות**: הזחה אומרת לפייתון מה "בתוך" בלוק ה-if

## רצף ההוראה

### שלב 1: משפט if פשוט
- להתחיל עם המקרה הפשוט ביותר: `if condition:`
- להשתמש בדוגמה קונקרטית: בדיקה אם סיסמה מתאימה
- להראות מה קורה כשהתנאי True לעומת False
- לשים לב ל: נקודתיים חסרות, אין הזחה, = לעומת ==

### שלב 2: הוספת else
- להציג את מושג ה"אחרת"
- להראות if-else כשני מסלולים: אחד או השני רץ
- לשים לב ל: הזחה של else ברמה לא נכונה

### שלב 3: תנאים מרובים עם elif
- לבנות מערכת ציונים: A, B, C, D, F
- להראות איך `elif` בודק בסדר ועוצר בהתאמה הראשונה
- להשוות למספר משפטי `if` (כולם נבדקים)
- לשים לב ל: סדר תנאים לא נכון (בדיקת < 60 לפני < 70)

### שלב 4: שילוב תנאים
- `and`: שניהם חייבים להיות אמיתיים
- `or`: לפחות אחד חייב להיות אמיתי
- `not`: הופך True ל-False ולהפך
- לשים לב ל: שימוש ב-"and" באנגלית במקום `and` של פייתון

### שלב 5: תנאים מקוננים
- if בתוך if אחר
- מתי לקנן לעומת מתי להשתמש ב-`and`
- לשים לב ל: קינון מוגזם (בדרך כלל אפשר לפשט)

## נקודות בדיקה

### אחרי שלב 1 (if)
לשאול: "מה ההבדל בין `=` ו-`==`?"
צפוי: `=` שם ערך במשתנה, `==` בודק אם שני דברים שווים

### אחרי שלב 2 (else)
לשאול: "אם תנאי ה-`if` הוא True, האם בלוק ה-`else` רץ?"
צפוי: לא, רק אחד מהם רץ

### אחרי שלב 3 (elif)
לשאול: "בבודק ציונים, למה צריך לבדוק A (≥90) לפני B (≥80)?"
צפוי: כי אם למישהו יש 95, שני התנאים אמיתיים, אבל אנחנו רוצים את ההתאמה הראשונה

### סוף השיעור
לשאול: "איך היית בודק אם מספר נמצא בין 10 ל-20?"
צפוי: `if number >= 10 and number <= 20:` או `if 10 <= number <= 20:`

## אם התלמיד מתקשה

- **אם מתקשה עם תנאים**: להשתמש באנלוגיות מהעולם האמיתי - "אם יורד גשם, קח מטרייה"
- **אם מתקשה עם סדר elif**: לצייר תרשים זרימה שמראה את מסלול ההחלטה
- **אם מתקשה עם and/or**: להשתמש בטבלאות אמת עם דוגמאות פשוטות
- **אם מקבל תוצאות לא צפויות**: להוסיף הדפסות לדיבוג כדי לראות איזה ענף רץ
