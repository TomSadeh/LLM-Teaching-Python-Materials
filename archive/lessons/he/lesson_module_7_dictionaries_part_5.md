## תוכן השיעור

### בהמשך לחלק 4

עכשיו אנחנו יכולים ליצור, לשנות, לגשת בבטחה, ולעבור בלולאה על מילונים. בואו נראה איך הם משמשים ביישומים אמיתיים.

### סטטיסטיקות דמות (סגנון RPG)

מילונים מושלמים לנתוני דמות במשחק:

```python
player = {
    "name": "{{hero}}",
    "level": 5,
    "health": 100,
    "inventory": ["wand", "potion", "{{item}}"],
    "skills": {"attack": 15, "defense": 10, "magic": 20}
}
```

### מילונים מקוננים

מילון יכול להכיל מילונים אחרים:

```python
school = {
    "{{house}}": {
        "points": 350,
        "head": "{{mentor}}",
        "members": ["{{hero}}", "{{friend}}"]
    },
    "Slytherin": {
        "points": 320,
        "head": "{{villain}}",
        "members": ["{{creature}}", "Pansy"]
    }
}

# גישה לנתונים מקוננים:
print(school["{{house}}"]["points"])  # 350
print(school["{{house}}"]["members"][0])  # {{hero}}
```

### מתי להשתמש ב-Dict לעומת List

| להשתמש ב-**List** כש... | להשתמש ב-**Dictionary** כש... |
|-------------------------|------------------------------|
| הסדר חשוב | צריך לחפש לפי שם/מפתח |
| הפריטים דומים | לכל פריט יש מזהה ייחודי |
| ניגשים לפי מיקום | לפריטים יש מספר תכונות |

```python
# List: אוסף מסודר של פריטים דומים
spells = ["{{spell1}}", "{{spell2}}", "{{spell3}}"]

# Dictionary: תכונות בעלות שם של דבר אחד
character = {"name": "{{hero}}", "age": 11, "house": "{{house}}"}
```

### תבנית מסד נתונים פשוטה

לאחסן רשומות מרובות באמצעות רשימה של מילונים:

```python
contacts = [
    {"name": "{{hero}}", "phone": "555-1234"},
    {"name": "{{friend}}", "phone": "555-5678"},
    {"name": "{{mentor}}", "phone": "555-9999"}
]

# למצוא איש קשר:
for contact in contacts:
    if contact["name"] == "{{hero}}":
        print(contact["phone"])
```

### לשים לב

- **קינון מוגזם**: יותר מדי רמות של קינון נהיה קשה לקריאה ותחזוקה
- **ערבוב תבניות גישה**: לזכור: dict משתמש במפתחות `["key"]`, list משתמש באינדקסים `[0]`

לשאול: "מתי היית משתמש במילון במקום ברשימה?"

## אם התלמיד מתקשה

- **אם מבולבל לגבי קינון**: להתחיל עם דוגמאות פשוטות. לצייר את זה: המילון החיצוני מכיל מילונים פנימיים, כמו תיקיות שמכילות קבצים.
- **אם לא בטוח מתי להשתמש ב-dicts לעומת lists**: לשאול "אכפת לך מהסדר, או שאתה רוצה לחפש דברים לפי שם?" סדר = list, חיפוש לפי שם = dict.
- **אם מוצף מהמורכבות**: להציע לפרק מבנים מורכבים לחלקים קטנים יותר קודם, ואז לשלב אותם.
