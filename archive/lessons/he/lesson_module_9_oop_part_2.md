## מושגי מפתח

| מושג | הסבר במשפט אחד |
|------|----------------|
| מילת מפתח `class` | איך מגדירים מחלקה חדשה בפייתון |
| מופע (Instance) | אובייקט ספציפי שנוצר מתבנית המחלקה |
| תחביר הנקודה | איך ניגשים לתכונות על אובייקט (למשל, `hero.name`) |

---

## תוכן השיעור

### בניה על חלק 1

למדנו למה מחלקות שימושיות - עכשיו בואו נראה איך ליצור אחת.

### המחלקה הראשונה שלך

המחלקה הפשוטה ביותר:

```python
class Character:
    pass

# יצירת מופע (אובייקט) מהמחלקה
hero = Character()
villain = Character()
```

### הוספת תכונות

אפשר להוסיף תכונות לאובייקט באמצעות תחביר הנקודה:

```python
hero = Character()
hero.name = "{{hero}}"
hero.health = 100

villain = Character()
villain.name = "{{villain}}"
villain.health = 80
```

### גישה לתכונות

```python
print(hero.name)        # מדפיס: {{hero}}
print(villain.health)   # מדפיס: 80
```

### תובנה מפתח: מחלקה לעומת מופע

- `Character` היא ה**מחלקה** (התבנית)
- `hero` ו-`villain` הם **מופעים** (אובייקטים ספציפיים)
- לכל מופע יש תכונות נפרדות משלו

```python
# זה מראה שהם אובייקטים שונים:
print(hero.name)      # {{hero}}
print(villain.name)   # {{villain}}
```

### לשים לב ל

- **סוגריים חסרים**: `Character` היא המחלקה, `Character()` יוצר מופע
- תלמידים שמנסים לגשת לתכונות על המחלקה עצמה במקום על מופע
- בלבול בין שם המחלקה לשמות משתנים

## אם התלמיד מתקשה

- **אם מחלקה לעומת מופע לא ברור**: להשתמש בדוגמאות מוחשיות - "כלב" היא המחלקה, "הכלב שלי פלאפי" הוא מופע
- **אם שוכח סוגריים**: להדגיש ש-`Character` היא כמו מתכון, `Character()` היא כמו לבצע את המתכון כדי להכין משהו
- **אם תחביר הנקודה מבלבל**: להשוות לאיך ניגשים לפריטים במילון עם סוגריים מרובעים, אבל אובייקטים משתמשים בנקודות
