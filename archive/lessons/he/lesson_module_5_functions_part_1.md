## תוכן השיעור

### הבעיה: קוד שחוזר על עצמו

להראות קוד שחוזר על אותו דפוס מספר פעמים:

```python
# ציור שלושה ריבועים - הרבה חזרות!
import turtle
t = turtle.Turtle()

# ריבוע 1
for i in range(4):
    t.forward(50)
    t.right(90)

t.penup()
t.forward(100)
t.pendown()

# ריבוע 2
for i in range(4):
    t.forward(50)
    t.right(90)

t.penup()
t.forward(100)
t.pendown()

# ריבוע 3
for i in range(4):
    t.forward(50)
    t.right(90)
```

לשאול: "מה קורה אם נרצה לשנות את גודל הריבוע? נצטרך לשנות בשלושה מקומות!"

### הפתרון: מתכון לשימוש חוזר

להציג את המושג: "פונקציה היא כמו מתכון שאפשר להשתמש בו כמה פעמים"

```python
# להגדיר את הפונקציה - ליצור את המתכון
def draw_square():
    for i in range(4):
        t.forward(50)
        t.right(90)

# לקרוא לפונקציה - להשתמש במתכון
draw_square()
t.penup()
t.forward(100)
t.pendown()
draw_square()
t.penup()
t.forward(100)
t.pendown()
draw_square()
```

### הפונקציה הפשוטה ביותר

להתחיל עם הפונקציה הכי פשוטה - בלי פרמטרים, בלי return:

```python
def say_hello():
    print("Hello!")
    print("Welcome to Python!")

# שום דבר לא קורה עד שאנחנו קוראים לה
say_hello()  # עכשיו היא רצה!
say_hello()  # אפשר לקרוא לה שוב!
```

להדגיש: `def` יוצר את הפונקציה, אבל הקוד שבפנים רץ רק כשקוראים לה.

### לשים לב ל

- **שכחת הסוגריים**: `say_hello` לעומת `say_hello()` - הסוגריים נדרשים לקריאה לפונקציה
- **שכחת הנקודתיים**: `def say_hello():` צריך נקודתיים בסוף
- **שגיאות הזחה**: כל הקוד בתוך הפונקציה חייב להיות מוזח

## נקודת בדיקה

לשאול: "מה ההבדל בין הגדרת פונקציה לקריאה לפונקציה?"
צפוי: הגדרה (עם `def`) יוצרת את הפונקציה אבל לא מריצה אותה. קריאה (עם סוגריים) באמת מריצה את הקוד שבפנים.

---

## אם התלמיד מתקשה

- **אם מבולבל לגבי def לעומת קריאה**: להשתמש באנלוגיה של ספר מתכונים - כתיבת מתכון בספר (def) לעומת בישול ממשי (קריאה)
- **אם שוכח לקרוא לפונקציות**: להוסיף רשימת בדיקה - "האם הגדרתי? האם קראתי?"
- **אם שגיאות הזחה**: להראות איך כל השורות בתוך הפונקציה חייבות להיות מוזחות באותה מידה
