# Exercise Narrative Template: Spec Implementation

**Usage:** ~18% of exercises
**Best For:** Complete_function with detailed Args/Returns specifications
**Pattern:** Requirements → Feature 1 → Feature 2 → Feature 3 → Testing

## Key Features
- Formal specifications
- Feature prioritization
- Test-driven approach
- Project delivery metaphor

## Structure
```
Opening (Project specifications)
Feature 1: HIGH priority
Feature 2: HIGH priority
Feature 3: MEDIUM priority
Feature 4: MEDIUM priority
Feature 5: LOW priority
Testing: Quality assurance
Closing (Project delivery)
```

## Priority Markers
- HIGH: Core functionality
- MEDIUM: Standard features
- LOW: Nice-to-have

## Theme Examples
- **dumblecode**: Ministry project assignment
- **chirthon**: Divine commission
- **compile-games**: Capitol requirements
- **pyfire**: Council specifications
- **pymentor**: Client requirements

## Use When
- complete_function with detailed specs
- Args/Returns documentation present
- Professional/formal tone appropriate
- Feature-based thinking
